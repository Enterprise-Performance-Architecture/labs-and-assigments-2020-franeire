Docker Tutorial

Docker is a platform that allows developers to organise and develop applications without the use of virtualisation. This is achieved by using whats known as containerisation. Containerisation is a type of virtualisation, but one based on the OS instead of the hardware.

Docker uses three key areas:
Dockerfile
This is a file that contains the commands used to create an image. Users can build these images using the Docker build command.
Images
The docker image is basically a template that can create containers.
Containers
These are the applications built using the docker images. Creating the Dockerfile is the first step in using docker. The command to do this is:

nano Dockerfile (it must have a capital D). The dockerfile used in this CA is below

FROM python:3
WORKDIR /
COPY index.html /test
CMD python -m http.server 8080


Docker build .
Once this has been run, the docker image has been created and can be viewed using the docker images command

Docker run
To build the container we based on our docker image, we need to run a command like the below:

docker run --rm -it --name my-docker-image -p 8080:8080 my-docker-image

This makes sure the container is interactive, names it "my-docker-image", uses port forwarding on 8080 and the image ID.

How I completed the CA

  • The first step was to create a folder called CA1, this would contain the Dockerfile and index.html
    • mkdir CA1
    • cd CA1
  • In the CA1 folder, I created my dockerfile, using nano Dockerfile and as described above, it had 4 lines of code and created a HTTP server on port 80, and copied one HTML file.
  • I then built my docker image using the below command:
    • docker build -t my-docker-image
  • I then built the container, using the below command:
    • docker run --rm -it --name my-docker-image -p 8080:8080 my-docker-image
  • I then exported it to the GitHub Repository under the CA1 folder
